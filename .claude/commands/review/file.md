# Claude 명령어: Review File

특정 파일 또는 디렉토리의 코드를 분석하여 버그, 품질, 보안 문제를 리뷰합니다.

## 사용법

```
/review:file <파일 또는 디렉토리 경로>
```

예시:

```
/review:file src/app/page.tsx
/review:file src/components/auth/
```

## 프로세스

1. `$ARGUMENTS`로 전달된 경로의 파일 또는 디렉토리 읽기
2. 코드 전체를 대상으로 아래 기준에 따라 리뷰

## 리뷰 기준

### 🐛 버그 / 잠재적 오류

- 로직 오류 및 잘못된 조건 분기
- Null/Undefined 참조 가능성
- 비동기 처리 누락 (await, Promise 핸들링)
- 엣지 케이스 미처리

### 🎨 코드 품질 / 가독성

- 네이밍 명확성 (변수, 함수, 컴포넌트)
- 중복 코드 및 불필요한 복잡도
- SOLID 원칙 위반
- 함수/컴포넌트 단일 책임 원칙

### 🔒 보안 취약점

- OWASP Top 10 기준
- XSS / SQL 인젝션 / CSRF
- 민감 정보 노출 (키, 토큰, 개인정보)
- 인증·인가 누락

## 출력 포맷

```
## 리뷰 요약
(전체 평가 한 줄)

## 🐛 버그 / 잠재적 오류
🔴 [심각] 파일명:라인 — 문제 설명
  → 수정 방법

🟡 [경고] 파일명:라인 — 문제 설명
  → 수정 방법

## 🎨 코드 품질 / 가독성
🟡 [경고] ...
🟢 [제안] ...

## 🔒 보안
🔴 [심각] ...

## ✅ 잘된 점
- ...
```

## 심각도 기준

| 심각도 | 기준 |
|--------|------|
| 🔴 심각 | 즉시 수정 필요 — 버그, 보안 취약점, 데이터 손실 가능 |
| 🟡 경고 | 수정 권장 — 잠재적 문제, 유지보수 어려움 |
| 🟢 제안 | 개선 제안 — 더 나은 방향이 있음 |

## 참고사항

- 지적 사항이 없는 섹션은 생략
- 각 문제에는 반드시 수정 방법 포함
- 파일명:라인 형식으로 위치 명시
